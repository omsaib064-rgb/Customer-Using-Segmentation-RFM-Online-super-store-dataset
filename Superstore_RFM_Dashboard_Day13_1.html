📋 SQL Data Cleaning & RFM Analysis
Complete SQL script: Table creation → Null checks → Data cleaning → EDA → RFM Base → Scoring (NTILE) → Segmentation → Business Insights
✅ Null Values Found
All 21 columns: 0 nulls in raw data. Dataset is clean. SQL null-check query confirmed. Discount zeros treated as valid (no discount applied).
📊 Duplicate Records
No row_id duplicates found. Multiple order lines per Order ID are valid (different products). 9,994 rows → 5,009 unique orders → 793 unique customers.
STEP 1–3: TABLE + NULL CHECK + DUPLICATES
-- Create Table CREATE TABLE superstore ( row_id INT PRIMARY KEY, order_id VARCHAR(20), order_date DATE, ship_date DATE, ship_mode VARCHAR(30), customer_id VARCHAR(15), customer_name VARCHAR(100), segment VARCHAR(30), region VARCHAR(20), category VARCHAR(30), sub_category VARCHAR(30), sales DECIMAL(12,4), quantity INT, discount DECIMAL(5,4), profit DECIMAL(12,4) ); -- Null Value Audit SELECT 'sales' AS col, COUNT(*) - COUNT(sales) AS nulls FROM superstore UNION ALL SELECT 'profit', COUNT(*) - COUNT(profit) FROM superstore; -- Result: 0 nulls in all columns ✅
STEP 6–8: RFM SCORING (NTILE + CASE WHEN)
-- RFM Base Calculation CREATE VIEW rfm_base AS SELECT customer_id, customer_name, segment, DATEDIFF('2017-12-31', MAX(order_date)) AS recency_days, COUNT(DISTINCT order_id) AS frequency, ROUND(SUM(sales), 2) AS monetary FROM superstore GROUP BY customer_id, customer_name, segment; -- RFM Scoring with NTILE(5) CREATE VIEW rfm_scores AS SELECT *, NTILE(5) OVER (ORDER BY recency_days DESC) AS r_score, NTILE(5) OVER (ORDER BY frequency ASC) AS f_score, NTILE(5) OVER (ORDER BY monetary ASC) AS m_score FROM rfm_base; -- Customer Segmentation Logic CASE WHEN r_score >= 4 AND f_score >= 4 AND m_score >= 4 THEN 'Champions' WHEN r_score >= 3 AND f_score >= 3 THEN 'Loyal Customers' WHEN r_score >= 4 AND f_score <= 2 THEN 'New Customers' WHEN r_score >= 3 AND m_score >= 3 THEN 'Potential Loyalists' WHEN r_score <= 2 AND f_score >= 3 AND m_score >= 3 THEN 'At Risk' WHEN r_score <= 2 AND f_score <= 2 THEN 'Lost' ELSE 'Need Attention' END AS customer_segment
📌 Business Insights from SQL Analysis
💡 Insight 1: Discount Destroys Profit
Orders with 31%+ discount generate negative profit. The 0% discount group generates $261K profit alone. Discount policy needs urgent review — especially for Furniture (Tables: -$18K loss).
💡 Insight 2: At-Risk Customers = $446K Revenue Danger
101 At-Risk customers once generated $446K but haven't bought recently. Re-engagement campaigns targeting these customers is the highest-ROI action available.
💡 Insight 3: West Region Most Profitable
West region: $725K sales, $108K profit (14.9% margin). Central region: $501K sales, only $39K profit (7.9% margin). Investigate pricing and discount strategy in Central.
